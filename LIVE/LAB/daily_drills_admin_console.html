Learner Flow

  • Daily menu: learner chooses a topic and drill session.
  • Launch: Daily passes video_id, origin_mode: daily_rounds, and Daily return URL.
  • Drills engine: learner completes questions and sees completion state.
  • Return: Daily-origin completion exposes Return to Daily.
  • Result: Daily reads mm_daily_drill_result and shows session-only completion.

Operator Boundaries

  • Can inspect Daily/Drills runtime filenames and the session handoff contract.
  • Can run manual QA using the checklist on this page.
  • Cannot save durable learner progress from this LAB console.
  • Cannot alter drill registry, Supabase rows, RLS, auth, WordPress redirects, Railway endpoints, or CDN objects.
  • Backend/admin save actions must be routed to MissionMed wiring authority.
